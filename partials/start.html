Getting Started

Installation procedure

Step 1 : get Curiosity

Curiosity has no external dependency. To use it, just clone the repository or download the archive from github.

Step 2 : customize the conf file

Your elasticSearch server may not on localhost ! Customize the conf.js file, at the root of Curiosity's folder :

//Conf.js

var globalConf = {

defaultServer : 'http://localhost:9200', // Curiosity will connect by default to this elasticsearch host

confServer : 'http://localhost:9200', // Curiosity stores its data (keywords, template, ...) on this server

confIndex : "curiosity-config", // Index of the ElasticSearch "confServer" where curiosity will store it's data

defaultConfDocumentType : "conf-doc", // Type of simple conf document

}

Be sure to follow json syntax, if not, the application will not start properly.

Step 3 : English or French ?

Curiosity is multilingual, right now the application is fully translated in English and French (almost).To change Curiosity's language, copy the file that you need from lang folder to txt.js (This process will be simplified soon).

curiosity/lang> ls

en.js fr.js

curiosity/lang> cp en.js ../txt.js
By default the application is in english (almost).

Step 4 : Enjoy and share your data !

After that, curiosity is ready. Open index.html with a recent browser and enjoy.

Note about Curiosity setting up

When you open index.html for the first time, curiosity will create its own index and store few conf files. It will take 1 to 10 second depending on your elasticsearch server, but it's better to wait a bit more. Wait during this period, then reload the page. The page looks like :

At the left of the navigation bar on the top of your screen, there is two buttons that represents tabs. The "Request" button is the main tab, and that's where you are. The "Customization" button allows to edit template and keyword (go on template and keywords documentation for example).

On the right side is the server selector, you may choose the server you will work on, but also add a new one by pressing the "new" button. A form will replace the select, where you can enter the new server address (do not forget the http://), then click the button "Connect" : if it works, the button will stay green and the server will be added to the server list. If not, the button will go red.

The second selector allows to choose the indexes (and alias). Pick one to query it !

The green/red light represents the curiosity's status. If it goes red, an error has occurred. If you want more information about it, check the developper console (a curiosity's console will soon be added).

First Query

Curiosity use elasticsearch's query string query syntax, so you can easily create complex queries. Click in the query form (first form in page center), enter your query then press "enter" or click on search button. If your request is syntaxly correct, the status light remains green and the result will appear at the bottom center of the page.

Here are some basics for building a query :

  • If you want to search a string on a specific field :

    the.path.to.my.fields:"string you search"
  • Getting docs that matchs a value range for a specific field :

    the.path.to.my.fields:[Start TO End]
